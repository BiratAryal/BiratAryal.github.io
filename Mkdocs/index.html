    📘 Setting Up and Deploying MkDocs Documentation

    Follow the steps below to set up your MkDocs documentation environment, customize your content, and deploy it to GitHub Pages.

    🚀 Step 1: Navigate to the Documentation Directory

    cd D:\ObsidianDocumentation\ObsidianDocumentation\documentations

    🐍 Step 2: Activate Python Virtual Environment

    venv\Scripts\activate

    If the virtual environment is not yet created, you can initialize it with:

    python -m venv venv

    Then activate it as shown above.

    📦 Step 3: Install Dependencies

    ✅ One-Time Setup (First Time Only)

    Create a requirements.txt file:

    pip freeze > requirements.txt

    📥 Installing Dependencies

    Install MkDocs Material:

    pip install mkdocs-material

    Install additional requirements:

    pip install -r requirements.txt

    📝 Step 4: Customize Your Documentation

    Make necessary content changes inside the docs/ directory based on your requirements.

    🛠️ Step 5: Build the Documentation Site

    mkdocs build
    • This command generates a static site inside the site/ directory.
    mkdocs serve

    - This command serves the static site from the md files.

    ☁️ Step 6: Deploy to GitHub Pages

    1. Create a new GitHub repository with the format:

      <your-username>.github.io

    2. Make the repository public.

    3. Copy the contents of the site/ folder into the root of your new GitHub repository.

    4. Push the files to GitHub.

    📄 Step 7: Configure .gitignore

    To avoid pushing unnecessary files, create a .gitignore file in your documentation root with the following content:

    venv/
site/

    🌐 Step 8: Access Your Website

    Once the files are pushed, your documentation will be live at:

    https://<your-username>.github.io
