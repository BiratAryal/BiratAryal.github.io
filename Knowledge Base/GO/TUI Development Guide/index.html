    Phase 1: Understanding Bubbletea & Lipgloss Architecture (Today)

    We'll begin with the fundamentals of how Bubbletea and Lipgloss work under the hood:

    📚 BUBBLETEA — Architecture Overview

    Bubbletea is based on The Elm Architecture, which is a pattern for managing state and effects in a functional, reactive way.

    🔁 Core Concepts

    Concept Description
    Model Holds the state of the application.
    Msg Represents events (user input, timers, commands).
    Init() Initializes the app and can return initial commands.
    Update() Handles Msgs, updates the Model, optionally returns a Cmd.
    View() Renders the UI as a string based on the current Model.
    Cmd Asynchronous tasks like HTTP requests, time.Sleep, etc.

    📘 Minimal Diagram:

    Text Only
    User Input / Events
        ↓
     Msg (message)
        ↓
    Update(model, msg)
        ↓
   New Model + Cmd (optional async)
        ↓
     View(model) → string → Rendered to Terminal

    You write:

    • Model: structure of your app’s state

    • Update: how the state changes when events come in

    • View: what the user sees based on the current state

    🎨 LIPGLOSS — Styling Your Terminal UI

    Lipgloss is like CSS for your terminal UI. It styles the string output (from View() in Bubbletea).

    🎨 Key Concepts:

    Concept Purpose
    lipgloss.NewStyle() Creates a new style chain.
    .Foreground(), .Background() Set text and background colors.
    .Bold(), .Italic(), .Underline() Text formatting.
    .Margin(), .Padding(), .Width() Layout helpers.
    .Render("text") Applies styles to text.

    You use Lipgloss inside the View() function in Bubbletea.

    🔧 Putting it Together: Mini Skeleton

    Go
    package main

import (
    "github.com/charmbracelet/bubbletea"
    "fmt"
    "os"
)

type model struct {
    count int
}

// Messages
type msg string

// Init
func (m model) Init() bubbletea.Cmd {
    return nil
}

// Update
func (m model) Update(msg bubbletea.Msg) (bubbletea.Model, bubbletea.Cmd) {
    switch msg := msg.(type) {
    case bubbletea.KeyMsg:
        switch msg.String() {
        case "q":
            return m, bubbletea.Quit
        case "up":
            m.count++
        case "down":
            m.count--
        }
    }
    return m, nil
}

// View
func (m model) View() string {
    return fmt.Sprintf("Counter: %d\n↑: up | ↓: down | q: quit", m.count)
}

// Main
func main() {
    p := tea.NewProgram(model{})
    if err := p.Start(); err != nil {
        fmt.Println("Error:", err)
        os.Exit(1)
    }
}
