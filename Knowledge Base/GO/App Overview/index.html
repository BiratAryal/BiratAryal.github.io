    üß± App Overview: What We'll Build

    A simple menu that lets the user choose an option using the arrow keys.

    • Press up/down to move the cursor.

    • Press enter to select.

    üß≠ Bubble Tea Architecture Overview

    We'll walk through each piece:

    Text Only
    1. Model (holds state)
2. Msg (represents events)
3. Init() (optional setup, like timers or Cmds)
4. Update() (handles logic, changes model, returns Cmd)
5. View() (renders UI from model)
6. main() (entry point)

    1Ô∏è‚É£ Model (üì¶ Application State)

    This struct holds the current state of the app: choices, cursor, and selected value.

    Go
    type model struct {
    cursor   int      // current highlighted item
    choices  []string // menu options
    selected string   // user's final choice
}

    üìå What is stored in state?

    • The current cursor position (which menu item is selected).

    • The list of choices to display.

    • The selected item (after pressing enter).

    2Ô∏è‚É£ Msg (üì® Messages or Events)

    We don‚Äôt need to define custom messages here because keyboard events come as tea.KeyMsg (a built-in type from Bubble Tea).

    You can think of this like:

    • "up" key ‚Üí move cursor up

    • "down" key ‚Üí move cursor down

    • "enter" key ‚Üí select the option

    Go
    // No need for custom message yet; we use built-in tea.KeyMsg

    3Ô∏è‚É£ Init() (üöÄ App Initialization)

    Runs once when the app starts. You can use it to return an initial Cmd (like a timer or async task). We don‚Äôt need that for now, so we return nil.

    Go

    return nil // no initial command needed
}

    4Ô∏è‚É£ Update (üîÅ Handles Events, Modifies State)

    This is the brain of the application. It receives Msg (like a keypress) and updates the model.

    Go
    func (m model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    switch msg := msg.(type) {
    case tea.KeyMsg:
        switch msg.String() {
        case "up":
            if m.cursor > 0 {
                m.cursor--
            }

        case "down":
            if m.cursor < len(m.choices)-1 {
                m.cursor++
            }

        case "enter":
            m.selected = m.choices[m.cursor]
            return m, tea.Quit // select and quit app
        }
    }

    return m, nil
}

    ‚úÖ What this does:

    • Arrow keys change the cursor position.

    • Enter key saves the selected item and exits the app.

    • Returns updated model and (optionally) a command (Cmd).

    5Ô∏è‚É£ View (üñºÔ∏è Renders UI from State)

    This function draws the screen based on the current state.

    Go
    func (m model) View() string {
    s := "What do you want to do?\n\n"

    for i, choice := range m.choices {
        cursor := " " // default
        if m.cursor == i {
            cursor = ">" // highlight current item
        }

        s += fmt.Sprintf("%s %s\n", cursor, choice)
    }

    if m.selected != "" {
        s += fmt.Sprintf("\nYou selected: %s\n", m.selected)
    }

    return s
}

    6Ô∏è‚É£ Main Function (üö™ Entry Point)
